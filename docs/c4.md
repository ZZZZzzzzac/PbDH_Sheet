# PbDH Sheet Framework C4 架构图

状态：Draft  
日期：2026-07-06

本文基于 `docs/PRD.md` 和 `docs/adr/` 画出 C4 Level 1-3。Level 4 Code 图暂不画；当前还没有实现代码，过早画类/函数层会变成伪精确。

## Level 1：系统上下文图

```mermaid
flowchart LR
  author["Person: Author（作者）<br/>制作 System Package，查看预览和校验错误"]
  player["Person: Player（玩家）<br/>导入 System Package，编辑、保存、导出角色"]
  host["External System: 静态托管平台<br/>提供 HTML / JS / CSS / PWA 应用壳"]
  framework["Software System: PbDH Sheet Framework<br/>无服务器 API 的静态 Web 应用<br/>把 System Package 渲染为 Sheet Tool"]
  package["Artifact: System Package<br/>zip 或目录包<br/>规则、资源、布局、样式、检查脚本、图片"]
  exports["Artifact: 导出文件<br/>Character Data JSON / HTML 快照 / 浏览器打印 PDF"]
  browserStorage[("Browser Storage<br/>IndexedDB / localStorage / PWA Cache")]

  author -->|"编写 / 上传 / 调试"| package
  package -->|"其他渠道分发<br/>QQ群 / GitHub 等"| player
  author -->|"查看 Author Preview / 校验错误"| framework
  player -->|"打开网页"| host
  host -->|"交付应用壳"| framework
  player -->|"上传本地 package"| package
  package -->|"导入"| framework
  player -->|"编辑角色卡 / 操作卡牌 / 运行检查"| framework
  framework -->|"保存缓存和角色数据"| browserStorage
  browserStorage -->|"读取缓存和角色数据"| framework
  framework -->|"导出"| exports
  exports -->|"备份 / 分享 / 打印"| player
```

## Level 3：Browser App 组件图

```mermaid
flowchart TB
  subgraph app["Browser App（React / TypeScript）"]
    appShell["app-shell<br/>模式切换、全局错误、PWA 状态"]

    subgraph packagePipe["System Package 管线"]
      packageLoader["package-loader<br/>zip/目录 -> 虚拟文件系统"]
      packageSchema["package-schema<br/>类型和结构解析"]
      packageValidator["package-validator<br/>fatal/error/warning/info/debug"]
      assetResolver["asset-resolver<br/>assetRef -> Blob URL / fallback"]
    end

    subgraph runtime["运行时核心"]
      stateStore["state-store<br/>Zustand 状态和 actions"]
      storageService["storage-service<br/>IndexedDB / localStorage 适配"]
      sheetRenderer["sheet-renderer<br/>Flow Layout + Module Registry"]
      moduleComponents["Sheet Module Components<br/>文本、长文本、资源、勾选、选择、图片、只读、卡牌"]
      dependencyEngine["dependency-engine<br/>view effects / data patches / option effects"]
      cardEngine["card-engine<br/>Card Definition + Card Instance 操作"]
    end

    subgraph validation["检查与导出"]
      validationRunner["validation-runner<br/>调度多个 JS checks"]
      exportPrint["export-print<br/>Print Mode / HTML 快照 / 图片等待"]
    end
  end

  packageInput["System Package zip / 目录包"] --> packageLoader
  packageLoader --> packageSchema
  packageSchema --> packageValidator
  packageValidator -->|"无 fatal/error 后"| stateStore
  packageValidator -->|"error diagnostics"| appShell

  stateStore --> sheetRenderer
  sheetRenderer --> moduleComponents
  moduleComponents -->|"dispatch action"| stateStore

  stateStore -->|"Character Data + Resource Libraries + Card State"| dependencyEngine
  dependencyEngine -->|"view effects / data patches / option effects"| stateStore

  stateStore -->|"Card Instance State"| cardEngine
  cardEngine -->|"card actions"| stateStore

  moduleComponents -->|"assetRef"| assetResolver
  assetResolver --> storageService
  stateStore -->|"debounce autosave"| storageService
  storageService --> stateStore

  stateStore --> validationRunner
  validationRunner -->|"启动 Web Worker"| worker["Validation Worker<br/>输入副本 -> issue list"]
  worker --> validationRunner
  validationRunner -->|"Validation Issues"| stateStore

  stateStore --> exportPrint
  assetResolver --> exportPrint
  exportPrint --> htmlSnapshot["只读 HTML 快照"]
  exportPrint --> browserPrint["浏览器打印 / 保存 PDF"]
```

## Level 3：关键边界放大图

### Dependency Engine 边界

```mermaid
flowchart LR
  character["Character Data<br/>字段值、选择值、卡牌实例状态"]
  resources["Resource Libraries<br/>资源条目和扩展字段"]
  rules["Dependency Rules<br/>sources / targets / when / actions"]
  engine["Dependency Engine<br/>单轮计算<br/>不读 DOM，不读 UI 临时状态"]
  view["View Effects<br/>显示/隐藏页面或模块、高亮、只读展示变化"]
  patch["Data Patches<br/>自动填充、设置勾选、清空字段等有限写入"]
  options["Option Effects<br/>筛选、包含、排除资源选项"]
  errors["Dependency Errors<br/>冲突或不支持的链式依赖"]
  store["State Store<br/>统一提交 effect/patch/error"]

  character --> engine
  resources --> engine
  rules --> engine
  engine --> view
  engine --> patch
  engine --> options
  engine --> errors
  view --> store
  patch --> store
  options --> store
  errors --> store
```

### Validation Script Runner 边界

```mermaid
flowchart LR
  store["State Store<br/>当前 Character Data / Resource Libraries / Card State"]
  input["检查输入副本<br/>structured clone + readonly guard"]
  runner["Validation Runner<br/>按顺序调度多个 checks"]
  worker1["Web Worker: check A<br/>只读输入 -> issues"]
  worker2["Web Worker: check B<br/>只读输入 -> issues"]
  normalize["Issue Normalizer<br/>校验 level/text/path/code"]
  report["Validation Report<br/>error / warning / info<br/>只提示，不硬阻止导出"]

  store --> input
  input --> runner
  runner --> worker1
  runner --> worker2
  worker1 --> normalize
  worker2 --> normalize
  normalize --> report
```

## 不画的 C4 图

- 不画 Level 4 Code 图：当前没有代码，类和函数名会过早锁死。
- 不画服务器端容器：第一版没有服务器 API，只有静态托管平台。
- 不画数据库服务：IndexedDB/localStorage 是浏览器本地存储，不是独立后端数据库。
